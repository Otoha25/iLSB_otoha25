# iLSB (TypeScript版)

従来版iLSBのファイル群が肥大化してきたため、依存関係を明確にし、副作用が起きにくくするために、TypeScript版のiLSBの開発を始めました。

## ディレクトリ構成

- `src/` ... ソースコードの置き場です。基本的にはここを編集することになります
  - `content-scripts/` ... ブラウザの全ページに埋め込まれて動作するプログラム
  - `repository/` ... Keyword Repositoryのプログラム
  - `tree` ... Question Treeのプログラム
  - `manifest.json` ... 拡張機能のマニフェストファイル
- `dist/` ... 成果物が生成されるところです。`npm run build`コマンドで`src`ディレクトリ内のコードがビルドされ、ここに配置されます

## 技術選定

### TypeScript

プロジェクトの規模が大きくなると、用意されている関数を全て把握するのは難しくなります。
JavaScriptは型が明示的でないため、関数の呼び出し側が間違った型の引数を与えてしまっても、実行するまで気づくことができません。
そのため、明示的に型を宣言し、エディタによる型チェックの機能を活用できるようにするために、TypeScriptの導入を行いました。

### Webpack

従来のiLSBは、複数のJavaScriptファイルがHTML内でimportされていて、呼び出し元の関数を探すことは容易ではありません。
さらに、全てのグローバル変数や関数に他ファイルからもアクセスできてしまうため、適切に情報隠蔽をする必要があります。
Webpackを使用することで、関数の他ファイルへのexportや、他ファイルからのimportを実現することができます。
